# 使用帮助

欢迎使用基估宝！这是一个专注于实时基金估值、持仓管理与投资追踪的开源工具。本指南将帮助你快速上手并掌握各项高级功能。

基估宝官方项目地址：[https://fund.cc.cd](https://fund.cc.cd)

备用地址：[https://hzm0321.github.io/jgb-doc/](https://hzm0321.github.io/jgb-doc/)

## 1. 核心功能

### 1.1 基金添加与实时追踪

- **快速添加**：在页面顶部搜索框输入 6 位基金代码（如 `000001`）即可添加至列表。
- **实时估值**：系统直连金融接口，提供分钟级的盘中估值更新，涨跌幅一目了然。
- **重仓追踪**：点击基金卡片可展开查看该基金的前十大重仓股及其盘中表现，助你洞察波动来源。
- **关联板块**：支持展示基金追踪的关联指数或行业板块，全局把握市场走势。

### 1.2 持仓与收益分析

- **持仓设置**：在基金详情中输入持有份额与成本价，系统将自动计算：
  - **当日收益**：基于最新估值的即时盈亏估算。
  - **累计收益**：自持仓以来的总盈亏及收益率。
  - **持有金额**：当前的资产总值。
- **关联持仓**：如果你在多个分组中都添加了同一只基金，在“全部”或“自选”列表中会展示该基金的**汇总持仓**。系统会自动计算跨分组的加权平均成本和总份额，并标注 ![guide-1.png](/img/guide/guide-1.png) 图标，方便你从全局视角查看单只基金的总仓位。
- **多维度排序**：支持按估值涨跌幅、持仓收益额、收益率、持有金额等规则进行灵活排序。

### 1.3 交易管理

- **连涨连跌**：在基金卡片或列表中，你会看到类似![guide-2.png](/img/guide/guide-2.png)(连涨3天)的统计标志。这是基于基金历史净值趋势自动计算的，能够直观反映该基金近期的走势动量。
- **状态标签**：在基金代码（如 `#000001`）后面，你可能会看到以下小标签，用于快速识别基金的特殊状态：
  - **​**![guide-3.png](/img/guide/guide-3.png)：表示该基金有**进行中的交易**（如申购待确认、赎回待到账等）。当你记录了交易流水且尚未完成确认时，此标签会自动出现，提醒你关注交易进度。
  - **​**![guide-4.png](/img/guide/guide-4.png)：表示该基金已设置了**自动定投计划**且当前处于启用状态。看到此标签即可知该基金正在按计划自动定投中。
  - **​**![guide-5.png](/img/guide/guide-5.png)：表示该基金**今日净值已更新**。当基金公司公布了当天的净值后，此标签会出现，让你一目了然地知道哪些基金的数据是最新的。对于 QDII 等净值公布较晚的基金，系统会自动适配其确认天数。
- **交易管理**：支持记录真实的交易流水，自动更新持仓成本与份额。
- **交易日逻辑**：系统内置交易日日历，自动处理 T+1 确认逻辑，确保收益计算准确。
- **基金转换**：支持记录基金间的转换操作，简化调仓记录流程。

### 1.4 收益日历

- **历史回顾**：通过直观日历视图查看过去每一天的盈亏情况，支持按月统计。

## 2. 高级特性

### 2.1 截图/OCR 导入

- **批量添加**：如果你有大量持仓，只需上传支付宝、天天基金等平台的持仓截图，系统将利用 OCR 技术自动识别基金代码、份额及成本，实现“一键搬家”。

### 2.2 自动定投计划

- **定投模拟**：支持设置日、周、月等周期性的定投计划，系统将自动记录定投流水，助你坚持长期投资。

### 2.3 分组管理

- **场景化分组**：支持创建多个分组（如“我的实盘”、“自选观察”等），不同分组独立统计收益，管理更清晰。

### 2.4 云端同步

- **多端联动**：登录邮箱账号后，数据可跨设备实时同步（PC、手机、平板），再也不怕更换设备。注意：同一时间段仅允许一台活跃设备往云端同步。当同步时出现“其它设备登录提示”弹框，点击“立即接管”可将当前设备设置为活跃设备。

## 3. 设置与个性化

### 3.1 基础设置

- **刷新频率**：可在 30秒至 300秒间自由调整，或输入自定义秒数，平衡数据实时性与流量消耗。
- **页面宽度**：PC 端支持通过滑块在 600px - 2000px 范围内调整页面宽度，适配不同分辨率 of 显示器。
- **组件开关**：可独立控制大盘指数、分组内搜索框在 PC/移动端的显示与隐藏。

### 3.2 排序个性化

- **形式切换**：排序入口支持“按钮”或“下拉单选”两种模式。
- **规则定制**：
  - **拖拽排序**：调整不同排序规则（如估值、持仓收益、持有金额）的优先级。
  - **别名编辑**：支持为排序规则设置别名（例如将“估算涨幅”重命名为“涨跌幅”），让界面更符合直觉。
  - **启用开关**：按需开启或关闭特定的排序逻辑。

### 3.3 大盘指数自定义

- **自选指数**：从预设列表中勾选你关注的指数（如上证指数、纳斯达克、恒生指数等）。
- **显示排序**：通过拖拽调整指数在顶部的展示顺序。

### 3.4 PC 表格高级配置

- **列管理**：支持显示/隐藏特定数据列，或通过拖拽调整列的左右顺序。
- **列固定（冻结）**：支持将特定列（如基金名称）固定在左侧，方便在大表格中横向滚动查看。
- **全名展示**：一键切换是否展示完整的基金名称，避免长名称被截断。
- **跨分组同步**：支持将当前的表格配置（顺序、显示隐藏、固定状态）一键同步到其他基金分组，无需重复设置。

### 3.5 移动端表格高级配置

- **个性化设置抽屉**：点击列表头左侧的“设置”图标，可唤起底部抽屉进行配置：
  - **展示完整基金名称**：由于屏幕宽度限制，默认会截断长基金名，开启此项可展示全名（配合自动缩放字体）。
  - **表头展示与排序**：支持隐藏不关注的列（如近1年收益、持仓成本等），并可通过拖拽调整各列 of 显示顺序。
  - **跨分组同步**：同样支持将当前分组的表格布局配置一键同步至其他所有分组。
- **编辑与批量操作**：
  - **进入编辑模式**：在默认排序模式下，点击表头的“铅笔”图标或**长按**任意基金行，即可进入批量编辑状态。
  - **批量移动/删除**：在编辑模式下，可勾选多只基金，一键移动到其他分组或进行批量删除。
  - **快速置顶**：编辑模式提供“移到最前”按钮，方便快速调整核心关注基金的顺序。
  - **手动拖拽排序**：在编辑模式下，可通过右侧的拖拽手柄自由调整基金的上下位置。

### 3.6 数据管理

- **导入导出**：支持将所有配置与数据导出为 JSON 文件进行本地备份，或通过导入功能实现数据迁移。
- **防息屏模式**：提供“保持屏幕常亮”功能，适合将旧手机/平板作为行情监控副屏。
- **主题切换**：支持亮色/暗色模式，并可与系统主题自动同步。

---

## 4. 数据隐私与安全

- **本地优先**：数据默认存储在浏览器的 `localStorage` 中，不经过后端服务器。
- **安全同步**：云端同步基于 Supabase 提供的安全认证，确保你的财务数据仅你自己可见。

<details className="faq-accordion" open>
  <summary>为什么盘中“估值”和晚间公布的“最终净值”有偏差？</summary>
  <div className="faq-content">
    <p>盘中实时估值是基于基金最新公布的<b>前十大重仓股</b>盘中走势推导计算的理论估值，仅供盘中投资决策与趋势跟踪参考。最终实际偏差产生的原因通常包括：</p>
    <ul>
      <li><b>隐形持仓影响</b>：基金财报仅定期披露前十大重仓股（约占仓位 40%~60%），其余非重仓股波动无法实时推算。</li>
      <li><b>日内调仓变化</b>：基金经理可能在交易日内进行了调仓换股或增减仓位。</li>
      <li><b>大额申赎与费率</b>：当计入申赎款项流动及日计提管理费用后，最终净值会有微调。</li>
    </ul>
    <p>💡 <i>提示：晚间基金公司官方公布净值后，系统会自动标注 <b>今日净值已更新</b> 徽章。</i></p>
  </div>
</details>

<details className="faq-accordion">

<details className="faq-accordion">
  <summary>如何理解“全部”、“自选”、“自定义分组”及“汇总持仓”的逻辑关系？</summary>
  <div className="faq-content">
    <p>在最新版本中，系统全面支持了<b>分组持仓独立管理</b>，核心逻辑如下：</p>
    <ul>
      <li><b>全部与自选分组</b>：两者共用同一套底层持仓数据；且“全部分组”可以进行<b>独立持仓设置</b>。</li>
      <li><b>自定义分组</b>：所有新建的自定义分组（如“激进组合”、“稳健实盘”）均为独立的持仓数据互不干扰。</li>
      <li><b>关联持仓计算</b>：如果未在“全部分组”中设置某只基金的持仓，但该基金在多个自定义分组中存在持仓，系统会在“全部”中以<b>关联持仓</b>形式自动汇总并加权统计各自定义分组的数据。</li>
      <li><b>汇总分组</b>：以分组的维度全局统计各个独立持仓的数据 and 盈亏，关联持仓不会被重复计算在内。</li>
    </ul>
  </div>
</details>

<details className="faq-accordion">
  <summary>我的持仓数据和财务信息存储在哪里？会泄露吗？</summary>
  <div className="faq-content">
    <p>基估宝坚持<b>“本地优先 (Local-First)”</b>的隐私设计原则：</p>
    <ul>
      <li>默认情况下，所有添加的基金、持仓份额和成本均严格存储在你本机的浏览器 <code>localStorage</code> 中，不会上传至任何第三方服务器。</li>
      <li>当你启用云端多设备同步功能时，数据端到端加密存储于安全可靠的 Supabase 数据库中，仅凭你的个人账号验证访问，绝对保障个人财务隐私。</li>
    </ul>
  </div>
</details>
